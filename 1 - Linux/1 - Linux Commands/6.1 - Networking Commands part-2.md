### curl

```curl``` ka full form hai **Client URL**.

Ye command url ke through data transfer karta hai ek server se dusre server par. Ye internet se data fetch karta hai aur url par data send bhi karta hai.

Ye HTTP, HTTPS, FTP, SFTP, SCP, SMTP aur aur bhi bahut saare protocols support karta hai.

Iska main use hota hai:
- Web pages ko fetch karna.
- APIs test karna (GET, POST, PUT, DELETE requests).
- File download/upload karna.
- Headers aur responses dekhna.

Matlab: ```curl``` ek tarah se mini browser ki tarah hai jo command-line par kaam karta hai.

<br>

**Syntax**:
```
curl [options] [URL]
```

<br>

**Common Uses of curl**:

- **Simple GET request**:
```
curl http://www.example.com
```
Website ka HTML source code terminal me print karega.

- **Sirf headers dekhna**:
```
curl -I https://example.com
```
Output:
```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1256
Server: nginx
```
Ye HTTP response headers show karega.

- **Redirects follow karna**:
```
curl -L https://example.com
```
Agar ek site redirect hoti hai (HTTP → HTTPS), to ```-L``` us redirect ko follow karega.

- **File download karna**:
```
curl -O https://example.com/file.zip
```
File ko original naam ke sath download karega.

```
curl -o newname.zip https://example.com/file.zip
```
File ko custom naam ke sath save karega.

- **APIs test karna (GET request)**:
Syntax:
```
curl -X <METHOD> <URL>
```
```
curl -X GET https://api.github.com/users/puneet
```
```-X``` flag ```curl``` mein HTTP request ka method specify karne ke liye use hota hai.

API ka JSON response terminal me milega.

- **POST request bhejna (form data)**:
```
curl -X POST -d "username=puneet&password=12345" https://example.com/login
```
Yaha ```-d``` ka matlab hai data, jab bhi curl ke through data bhejna ho to ```-d``` flag ka use kiya jata hai.

Ye server ko form data send karega.

- **POST request with JSON data**:
```
curl -X POST -H "Content-Type: application/json" -d '{"name":"Puneet","role":"DevOps"}' https://api.example.com/users
```
Ye ek JSON payload API ko bhejta hai.

- **PUT request (update data)**:
```
curl -X PUT -H "Content-Type: application/json" -d '{"role":"Admin"}' https://api.example.com/users/1
```

- **DELETE request**:
```
curl -X DELETE https://api.example.com/users/1
```

- **Basic authentication ke sath request**:
```
curl -u username:password https://example.com
```

- **Bearer Token (JWT, OAuth, etc.) authentication ke saath request**:
```
curl -H "Authorization: Bearer your_token_here" https://api.example.com/data
```

- **Response headers + body ek sath dekhna**:
```
curl -i https://example.com
```

- **Verbose/debug mode**:
```
curl -v https://example.com
```
Request aur response ka detailed debug info milega.

### wget

```wget``` ka full form hai **World Wide Web Get**.

Ye ek command-line utility hai jo HTTP, HTTPS aur FTP protocols ke through files download karne ke liye use hoti hai.

```wget``` ka sabse bada fayda hai ki ye non-interactive tool hai → matlab background me bhi kaam kar sakta hai, chahe tum logged in ho ya nahi.

Ye resume, recursive download aur mirroring support karta hai.

Matlab: ```wget``` mainly file downloader hai, jabki ```curl``` ek general-purpose data transfer tool hai.

**Syntax**:
```
wget [options] [URL]
```

<br>

**Common Uses of wget**:

- **Simple file download**:
```
wget https://example.com/file.zip
```
File ko current directory me download karega.

- **File ko custom naam se download karna**:
```
wget -O newname.zip https://example.com/file.zip
```
File ko ```newname.zip``` naam se save karega.

- **Resume download (agar bich me ruk gaya ho)**:
```
wget -c https://example.com/largefile.iso
```
Yaha ```-c```=Continue hai. Download waha se continue hoga jaha se ruk gaya tha.

- **Background me download**:
```
wget -b https://example.com/file.zip
```
Yaha ```-b```=Background hai. File background me download hogi

### scp

```scp``` ka full form hai **secure copy**.

Ye command local aur remote systems ke beech file aur directories ko securly copy karne ka kaam karta hai.

Ye command basically SSH (Secure Shell) protocol ke upar bana hua hai.

Matlab: jab tum ```scp``` use karte ho, tumhari file transfer bhi encrypted channel ke through hoti hai → isliye secure hai.

**Syntax**:
```
scp [options] source destination
```

**Source aur Destination ke format**:
- Local to Remote:
```
scp file_name.txt username@remote_host_ip:/path/to/destination/
```
- Remote to Local:
```
scp username@remote_host_ip:/path/to/file.txt /local/path/
```
- Remote to Remote (ek server se dusre server tak, direct):
```
scp username1@host_ip_1:/path/file.txt username2@host_ip_2:/path/
```

**Common Options**:
- ```-r``` → recursively (agar directory copy karni ho), Agar directory ke ander aur directories hain to ye option sabhi ki copy karega.
- ```-P``` → remote host ka port number dena (default SSH port 22 hota hai, par kabhi kabhi custom hota hai jaise 2222).
- ```-p``` → file ki original permissions aur timestamps preserve karta hai.
- ```-C``` → compression enable karta hai (large files ke liye fast transfer), matlab file ko compress karke bhejta hai.
- ```-v``` → verbose output deta hai (debugging ke liye helpful), detail batata hai.

**Examples**:

**Example 1: Local file ko remote server par bhejna**:
```
scp notes.txt puneet@192.168.1.10:/home/puneet/
```
- Ye ```notes.txt``` file tumhare local system se ```192.168.1.10``` wale remote server ke ```/home/puneet/``` folder me copy karega.
- Jab command run karoge, tumse remote system ka password poocha jayega (SSH ka).

**Example 2: Remote server se file apne local system me lana**:
```
scp puneet@192.168.1.10:/home/puneet/report.pdf /home/localuser/Downloads/
```
Ye ```report.pdf``` ko remote server se copy karke tumhare local machine ke ```Downloads``` me daal dega.

**Example 3: Directory copy karna (recursive mode)**:
```
scp -r project puneet@192.168.1.10:/home/puneet/
```
Ye tumhare local ```project``` directory (uske andar ki sab files aur sub-directories ke saath) remote system pe copy karega.

**Example 4: Remote to Remote copy (directly ek server se dusre server pe)**:
```
scp user1@192.168.1.10:/home/user1/data.tar.gz user2@192.168.1.20:/home/user2/
```
Isme file directly ek remote server se dusre server pe transfer ho jaati hai. Tumhara local system bas initiate karta hai.

**Example 5: Custom SSH Port ke saath**:
```
scp -P 2222 backup.zip puneet@192.168.1.10:/home/puneet/
```
Agar remote server SSH ke liye default port 22 ki jagah 2222 use karta hai:

**Example 6: Permissions aur timestamps preserve karna**:
```
scp -p config.yaml puneet@192.168.1.10:/etc/myapp/
```
Ye ```config.yaml``` ko bhejega aur uski file permissions aur modified date same rakhega.

<br>

```scp``` **aur** ```rsync``` **ka difference**:
- ```scp``` → simple file copy kar deta hai, har bar puri file copy karega.
- ```rsync``` → smart hai, agar file already hai to sirf differences (delta changes) copy karega. Large data aur repeated transfer ke liye ```rsync``` fast aur efficient hota hai.
